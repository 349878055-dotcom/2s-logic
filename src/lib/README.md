# 2S-LOGIC | lib 逻辑总部索引 (二级)

> **🎯 区域职责：** 本大区是项目的「最高逻辑大脑」。全盘负责 11 维度人格 OS 的演化计算、意图解析、全局状态流转与数据持久化。

## 1. 核心脑区导航 (Folders)
本大区内的三个子车间严禁互相越权操作，进入前必读本说明：

- **[/engines] - 动力与计算核心**
  - 范围：$C_{n+1}$ 演化算法 (`engine_evolving.ts`)、意图识别 (`will_engine.ts`) 等。
  - 禁令：这里只负责“算数学题”。严禁在此处直接读取数据库或操控前端 UI。算完的结果必须交出去。
- **[/core] - 系统骨架与血液**
  - 范围：全局类型定义 (`types.ts`) 和 Zustand 状态树 (`store.ts`)。
  - 禁令：`types.ts` 是演化底座的“基因链”，严禁 AI 擅自增删 11 维度的基础属性；修改 `store.ts` 时必须极度谨慎，防止状态死锁。
- **[/vault] - 存储与吞吐金库**
  - 范围：文件/数据库的 I/O 读写动作 (`io_engine.ts`)。
  - 禁令：这里是个“搬运工”，只负责把数据安全地存进硬盘。严禁在这里写任何业务计算逻辑！

## 2. ⚠️ 绝对隔离禁令 (CRITICAL)
- **纯净无尘室：** 本大区是绝对的「逻辑真空区」。**严禁引入任何 React 组件 (Components)、严禁写任何 HTML/JSX/Tailwind CSS 样式！**
- **原子化铁律：** 本区内的所有代码修改，必须严格遵守目标文件头部的【ATOM INDEX】（原子索引）。禁止为了图省事把多个函数合并成一坨黑盒。
